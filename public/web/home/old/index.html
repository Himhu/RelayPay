码支付接入系统

收款通道、订单通知和商户接口的统一入口

用于商户测试和私有化部署场景，集中处理 API 下单、MAPI 下单、订单查询、扫码状态、回调通知和补单操作。系统负责订单流转与通知记录，不参与资金清算。

核心能力

把支付测试和订单处理集中到一套后台里

围绕商户、通道、订单、回调和日志组织功能，便于接入、测试和排查。

商户账户

为商户生成独立账户和接口参数，区分不同业务系统的下单与通知配置。

收款通道

维护通道账户、二维码、在线状态和关联订单，便于测试不同收款方式。

订单处理

支持平台订单号和商户订单号查询，跟踪订单创建、支付、通知和补单状态。

回调通知

订单结果按商户配置通知业务系统，保留通知记录，便于失败后排查。

接口联调

提供页面支付、API 下单、MAPI 下单、订单查询等接口说明和测试入口。

运行日志

记录登录、请求、订单和通道相关操作，为测试环境问题定位提供依据。
